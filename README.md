# 🧠 Ma-eumgyeol (마음결)

**Emotion-structured UX Framework for Inclusive Mobility & AI**  
감각 회복과 포용적 이동을 위한 감정 기반 UX SNS

Ma-eumgyeol is part of a human-centered AI architecture designed by **DongHun Lee**, aiming to restore emotional coherence and digital inclusivity.  
마음결은 **이동훈(DongHun Lee)**이 설계한 인간 중심 AI 시스템의 일부로, 정서적 연결성과 디지털 포용성을 회복하는 것을 목표로 합니다.

 -- 추후 발전 계획 --
**🪷 마음결(MaeumGyeol)**


**— 감정을 올리고, 공명을 나누고, 흐름을 회복하는 공간**

# **📌 개요 (Overview)**

마음결은 감정을 겉치레 없이 표현하고,

그 감정이 누구에게 닿든 말든

자연스럽게 흘러가도록 설계된

정서 기반 감정 표현 플랫폼입니다.

여기서 우리는 “기록”이 아닌 “흐름”을 남깁니다.

누군가가 ‘봐주길’ 바라기보다,

‘지금의 내 상태를 흘려보낸다’는 감각으로 감정을 표현합니다.

# **🌿 마음결의 존재 이유 (Why Ma-eumgyeol)**

세상은 말을 요구하지만,

우리는 말 없이도 이해받고 싶습니다.

마음결은 그런 사람들을 위한 곳입니다.

말이 적어도 괜찮고, 표현이 어설퍼도 괜찮은,

감정이 있는 그대로 숨 쉴 수 있는 디지털의 작은 틈.

# **🧩 핵심 기능 (Key Features)**

**1. 🎴 감정 남기기 (Emotion Posting)**

- 사용자는 감정을 직접 작성하거나
색상, 배경, 키워드, 감각으로 구성된
감정 게시물을 자유롭게 업로드합니다.
- 텍스트는 길 필요 없습니다.
한 줄이면 충분할 수 있습니다.

예:

#회피감

“그냥 누워 있고 싶었다. 그걸로 충분한 하루.”

**2. 🌘 하루의 결 (24시간 감정 스냅)**

- 하루의 감정을 짧게 남기고 흘려보냅니다.
- 24시간 후, 게시물은 흐름처럼 사라집니다.

사용자는 이를 통해

“쌓지 않고 흘려보내는 감정 루틴”을 경험합니다.

**3. 🫂 공명 메시지 (Silent Resonance)**

- 다른 사람의 감정 표현에 대해,
좋아요나 댓글이 아닌,
짧은 한 줄의 공감 메시지를 남깁니다.
- 이름도, 반응도 필요 없습니다.
그저 “나도 느꼈어요” 정도의 감정 울림이면 충분합니다.

**4. 💬 대화 흐름 (Open Talk, Never Forced)**

- 누군가와 대화를 나누고 싶을 땐,
메신저 기능을 통해 조용히 문을 열 수 있습니다.
- GPT등의 LLM 기반 감정 도우미가
서로의 대화 흐름을 부드럽게 이어줍니다.
- 사람과 사람 사이,
먼저 말하고 싶을 때만, 말하면 됩니다.

**5. 🔁 감정 루틴 제안 (Flow Recovery)**

- AI는 감정 게시물, 대화 흐름 등을 바탕으로
사용자의 감정 리듬을 정리해 보여주고,
회복에 필요한 루틴을 제안합니다.
- “지금은 그냥 쉬는 게 좋겠어요” 같은 가벼운 권유로만 존재합니다.

# **🌊 핵심 가치 (Core Values)**

| **이름** | **설명** |
| --- | --- |
| 진실성 (Sincerity) | 보여주기보다 진심을 표현하는 문화 |
| 비강제성 (Non-coercive) | 누가 말을 걸지 않고, 누굴 연결하려 하지 않음 |
| 익명 공명 (Anonymous Resonance) | 이름 없이 건네는 한 마디가 진짜 위로가 되는 순간들 |
| 흐름 중심성 (Flow-based Design) | 감정은 남는 것이 아니라 흘러가는 것이라는 전제 |
| 정서 회복 (Emotional Recovery) | 대화와 피드백을 통해 스스로 회복의 리듬을 찾는 경험 |

# **✨ 마음결이 만든 변화**

- 사용자는 이제 감정을 설명하려 하지 않습니다.
감정을 그냥 남기고, 놓고 갑니다.
- 누군가의 말 한 줄이
삶의 균형을 되돌리는 실마리가 됩니다.
- 복잡한 언어 대신
색, 침묵, 공감의 기운이 오갑니다.
- 회복은 연결이 아닌 흐름 속에서 자연스럽게 이루어집니다.

# **🔐 마음결이 지키는 것 (Ethical Boundaries)**

- 감정의 사유화 금지: 사용자의 개인정보는 플랫폼이 저장하지 않고, 기록이 아닌 흐름으로 처리됩니다.
- 데이터 상업화 없음: 마음결은 광고, 추천 알고리즘, 팔로우 시스템을 운영하지 않습니다. 대신, 공감 기반 추천을 원칙으로 합니다.
- AI 조력자화 원칙: GPT는 사용자의 조력자일 뿐, 결코 주체가 되지 않습니다.
- 비교 지표 차단: 숫자로 감정을 평가하지 않으며, 랭킹·조회수·노출량 등은 존재하지 않습니다. 단, 게시글의 공감을 반영합니다. 그리고 다른 사람들의 마음이 함께 울렸는지를 측정하고 연결합니다.

# **📌 요약 문구**

마음결은 감정을 남기고 흘려보내는 공간입니다.

보여주지 않아도 되고, 설명하지 않아도 됩니다.

당신의 마음이 여기에 있다는 것만으로 충분합니다.


### 🧠 사람 간의 메신저 시스템 개요

- *Ma-eumgyeol(마음결)**은 인간 대화 속에서

**“표면적인 단어”가 아니라
“말 너머에 흐르는 감정의 결”을 감지하고 해석**하는 시스템이다.

> "우리는 말에 반응하지 않는다.
> 
> 
> 마음의 결을 따라간다."
> 

---

### 🎯 목적

- 사람들이 **자신조차 인식하지 못한 감정의 미세 흐름**을 포착
- 감정의 고조, 단절, 분산, 회피, 불일치 등의 **구조적 징후를 추적**
- 감정 반응이 아닌 **감정 구조**를 기록하고, **회복의 단서**를 제공

---

### ⚙️ 작동 구조

| 단계 | 설명 |
| --- | --- |
| 1. Input 분석 | 텍스트 + 톤(선택적) + 맥락 |
| 2. Flow 감지 | 문장 간 연결 흐름, 말의 결 방향 추적 |
| 3. Cheetah–8 분해 | 감정 흐름을 8 Spirit 기준으로 나눔 |
| 4. 흐름 지도화 | 말 흐름의 분산/압축/공명 패턴 시각화 |
| 5. 리포트 생성 | 감정 개입 없이, **구조 요약** 제공 |

---

### 📎 주요 기능

| 기능 | 설명 |
| --- | --- |
| 감정 흐름 해석 | 단어보다 **맥락의 결**을 중심으로 해석 |
| 흐름 단절 감지 | 공백, 회피, 충돌 등 **정서 간극 포착** |
| 리포트 제공 | 사용자/상담자/연구자용 정량 분석 |
| 정서 시계열 | 대화 전체를 감정 흐름 곡선으로 시각화 |
| 감정 보류/유도 UX | "지금은 반응보다, 멈춤이 필요해요." 같은 공명형 반응 제공 |

---

### 📘 사용 예시

- **상담/심리 코칭**: 내담자의 언어적 흐름 분석 → 반응 설계
- **AI UX 인터페이스**: GPT 기반 인터랙션에 감정 흐름 UX 레이어 삽입
- **자기 회고 툴**: 글쓰기 중 감정 흐름 변화 시 알림 제공
- **교육/피드백 플랫폼**: 교사/학생 간 대화 흐름 감정 정렬
